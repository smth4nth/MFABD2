# 参与开发

## 1. 💡 外部贡献指南
如果您有意向参与代码贡献，请参考以下流程：

1. **提交 PR 至公测流**：请基于 `main` 分支创建您的功能分支，并在完成后向本仓库的 `develop` 提交 Pull Request。
2. **署名**：我们高度关注贡献者署名。当您的代码在公测版验证通过后，**维护者会手动将该功能分支并入 `main`**。
3. **自动识别**：只要保留了您的 Commit 记录，GitHub 就会自动将您列入仓库主页的 `Contributors` 名单。

> **关于分支数量的提示**：
> 仓库中存在较多 `feat/*` 分支是正常的。这是为了实现“多线程并发开发”并确保主链不被未成熟的代码污染。我们欢迎您在这些分支中寻找灵感或提交修复。

### 核心开发规范
* **独立分支**：所有改动请基于 `Main` 拉取生成独立分支，严禁散`提交`直接推送到 `Main` 或 `Develop`。
* **保留轨迹**：合并时必须带上 `--no-ff` 参数。我们禁止变基（Rebase）或压缩合并，以确保每一条功能演进的脉络清晰可追溯。
* **回归路径**：功能在 `Develop` 验证稳定后，再由该功能分支发起 PR 指向 `Main` 分支。
