# 参与开发

## 1. 💡 外部贡献指南

欢迎提交 Pull Request！流程如下：

1. **创建功能分支**：从 `main` 拉取并创建您自己的功能分支（如 `feat/xxx`），
   开发完成后直接向本仓库的 `main` 分支提交 Pull Request。
   **切勿直接向 `main`、`develop`、`alpha` 推送代码。**
2. **审核与合并**：维护者会 Review 您的 PR，通过后由维护者合并。
   合并后您的 Commit 记录将保留，GitHub 会自动将您列入 `Contributors` 名单。

> **关于分支数量的提示**：
> 仓库中存在较多 `feat/*` 分支是正常的。这是为了实现"多线程并发开发"
> 并确保主链不被未成熟的代码污染。欢迎在这些分支中寻找灵感或提交修复。

### 核心开发规范

- **独立分支**：所有改动请基于 `main` 拉取生成独立分支，严禁散`提交`直接推送到 `main` 或 `develop`。
- **保留轨迹**：合并时必须带上 `--no-ff` 参数。我们禁止变基（Rebase）或压缩合并，以确保每一条功能演进的脉络清晰可追溯。
- **回归路径**：功能在 `develop` 验证稳定后，再由该功能分支发起 PR 指向 `main` 分支。
